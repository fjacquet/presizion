### Integration Gaps (cosmetic, non-blocking)

**Gap 1: UTIL-03 — Utilization factor missing from Step 2 formula display string**

- File: `src/components/step2/ScenarioResults.tsx` lines 40–45
- The `cpuFormulaString()` call does not pass `cpuUtilizationPercent`. When a user enters CPU utilization%, the formula string shows `ceil(2000 × 1.2 / 4 / 48)` instead of `ceil(2000 × 70% × 1.2 / 4 / 48)`.
- **Calculation is correct.** Only the display formula string is incomplete.
- Fix: add `cpuUtilizationPercent: currentCluster.cpuUtilizationPercent` to the `cpuFormulaString` call parameters.

**Gap 2: PERF-03 — SPECint mode shows duplicate count in ScenarioResults**

- File: `src/components/step2/ScenarioResults.tsx` lines 71 and 91
- In SPECint mode, both the "CPU-limited" row and the "SPECint-limited" row display `result.cpuLimitedCount` (same value, because constraints.ts reuses the cpu slot for specint). Confusing visually.
- **Value is numerically correct.** Display design needs the "CPU-limited" 3-column row hidden or relabeled when `sizingMode === 'specint'`.
- Fix: conditionally hide or relabel the 3-column CPU-limited row when `sizingMode === 'specint'`.

### Dead Code (tech debt, non-blocking)

**`src/lib/display/formulaStrings.ts` — orphaned runtime module**

- Contains `getCpuFormulaString`, `getRamFormulaString`, `getDiskFormulaString`, `getSpecintFormulaString` (all with `= N servers` suffix)
- Only consumed by its own test file; never imported by any UI component
- UI uses parallel `src/lib/sizing/display.ts` module (same-named functions, different suffix format)
- Risk: Maintenance confusion — two implementations of the same concern; tests assert a format users never see
- Resolution: Either wire `formulaStrings.ts` as the canonical display module and remove `sizing/display.ts`, or delete `formulaStrings.ts` and its tests and update CALC-07 coverage to reference `sizing/display.ts` only.
