<feature>
  <name>SPEC Lookup Service — Slug Derivation + Fetch</name>
  <files>src/lib/utils/specLookup.ts, src/lib/utils/__tests__/specLookup.test.ts, src/lib/config.ts</files>
  <behavior>
    ## Slug Derivation (cpuModelToSlug)

    The slug algorithm (from spec-search convert_csv.py):
    1. Lowercase the input
    2. Strip noise: "(r)", "(tm)", "cpu", "processor", clock speed patterns like "@ 2.40ghz", core count patterns like "96-core"
    3. Replace all non-alphanumeric sequences with a single hyphen
    4. Trim leading/trailing hyphens

    Test cases:
    - "Intel(R) Xeon(R) Gold 6526Y CPU @ 2.40GHz" -> "intel-xeon-gold-6526y"
    - "AMD EPYC 9654 96-Core Processor" -> "amd-epyc-9654"
    - "Intel(R) Xeon(R) Platinum 8480+" -> "intel-xeon-platinum-8480"
    - "" -> "" (empty input returns empty)
    - "  Intel Xeon E5-2680 v4  " -> "intel-xeon-e5-2680-v4"

    ## Fetch Service (fetchSpecResults)

    - Fetches `{SPEC_SEARCH_API_URL}/data/processors/{slug}.json`
    - The JSON response shape: `{ processor: string, results: Array<{ benchmark, vendor, system, baseResult, peakResult, cores, chips, ... }> }`
    - Filters results to only `benchmark === 'CINT2017rate'`
    - Returns `SpecLookupResult` with:
      - `results: SpecResult[]` (filtered benchmark results)
      - `status: 'ok' | 'no-results' | 'error'`
    - On network error or non-200 response: returns `{ results: [], status: 'error' }`
    - On 200 but no CINT2017rate matches: returns `{ results: [], status: 'no-results' }`

    ## Types

    ```typescript
    interface SpecResult {
      vendor: string;
      system: string;
      baseResult: number;
      peakResult: number;
      cores: number;
      chips: number;
    }

    interface SpecLookupResult {
      results: SpecResult[];
      status: 'ok' | 'no-results' | 'error';
    }
    ```

    ## Config Constants

    ```typescript
    export const SPEC_SEARCH_API_URL = 'https://fjacquet.github.io/spec-search'
    export const SPEC_SEARCH_WEB_URL = 'https://fjacquet.github.io/spec-search'
    ```
