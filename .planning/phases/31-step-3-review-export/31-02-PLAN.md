<interfaces>
<!-- Key patterns the executor needs -->

Recharts ResponsiveContainer accepts height as number (pixels) or string ("100%").
The downloadChartPng function takes a ref to the div containing the SVG -- this ref must stay on the innermost div wrapping the chart SVG.

Current pattern (all 4 charts):
```tsx
<div ref={someRef}>
  <ResponsiveContainer width="100%" height={300}>
```

Target pattern:
```tsx
<div className="h-48 sm:h-72">          {/* sizing div */}
  <div ref={someRef} className="h-full"> {/* ref div for PNG capture */}
    <ResponsiveContainer width="100%" height="100%">
```

CRITICAL: The ref div MUST have `className="h-full"` to propagate height from the outer div to ResponsiveContainer. Without it, height="100%" resolves to 0px (Pitfall 3 from research).

CapacityStackedChart special case: Uses dynamic heights (`showStorage ? 220 : 130`). Replace with a wrapper `<div className="h-[130px] sm:h-[220px]">` for the HCI case and `<div className="h-[130px]">` for disaggregated.
</interfaces>
