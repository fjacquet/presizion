  <behavior>
    Step 1 — Extend ClusterImportResult (src/lib/utils/import/index.ts):
    Add three optional fields to ClusterImportResult:
      detectedScopes: string[]
        - Array of unique scope keys. Key format: "${datacenter}||${cluster}" when both columns present; "${cluster}" when only cluster present; single-element array ["__all__"] when neither column is in the file.
        - Always populated (never undefined) — parsers set it, defaults to ["__all__"] if no scope columns found.
      scopeLabels: Record<string, string>
        - Human-readable label per scope key.
        - Format: "CL-A (DC1)" when both datacenter and cluster present; "CL-A" when cluster only; "All" for "__all__".
      rawByScope: Map<string, Omit<ClusterImportResult, 'sourceFormat' | 'detectedScopes' | 'scopeLabels' | 'rawByScope'>>
        - Per-scope aggregated numeric data. Each value is the aggregate of rows belonging to that scope.
        - For ESX-level fields (totalPcores, existingServerCount, etc.) on the "__all__" scope, use the overall calculation. For per-scope raw values, omit ESX fields (they cannot be split per scope from this data).
        - Mark as optional in the interface (?: Map<...>) to avoid JSON-serialization issues downstream; parsers always populate it.

    Step 2 — Add alias maps to columnResolver.ts:
    Add two new exported constants:
      CLUSTER_ALIASES: ColumnAliasMap = {
        cluster_name: ['Cluster', 'cluster', 'ClusterName', 'cluster_name', 'Cluster Name'],
      }
      DATACENTER_ALIASES: ColumnAliasMap = {
        datacenter_name: ['Datacenter', 'datacenter', 'DC', 'DataCenter', 'data_center'],
      }
    No changes to resolveColumns() — the new maps are passed in with empty required sets.

    Step 3 — Update rvtoolsParser.ts:
    In parseRvtools(), after building colMap with existing RVTOOLS_ALIASES, also resolve cluster and datacenter columns using CLUSTER_ALIASES and DATACENTER_ALIASES (empty required sets — both optional).
    During the row-iteration loop, for each non-template VM row:
      - Read cluster value: str(row[colMap_cluster['cluster_name']]) or '' if absent
      - Read datacenter value: str(row[colMap_dc['datacenter_name']]) or '' if absent
      - Compute scopeKey = dc && cluster ? `${dc}||${cluster}` : cluster ? cluster : '__all__'
      - Accumulate per-scope totals into a Map<string, { totalVcpus, totalMemMib, totalDiskMib, vmCount }>
    After the loop, build detectedScopes, scopeLabels, and rawByScope:
      - detectedScopes = [...scopeMap.keys()] (insertion order preserved)
      - scopeLabels: for each key, parse back: if key.includes('||') → "cluster (dc)"; else if key === '__all__' → 'All'; else → key
      - rawByScope: for each scope entry, compute totalDiskGb/avgRamPerVmGb the same way as the main aggregate
    Return these three fields added to the existing return object.
    The top-level aggregate (totalVcpus, totalVms, etc.) remains ALL rows — unchanged behavior.

    Step 4 — Update liveopticParser.ts:
    Apply the same per-scope accumulation approach to the aggregate() function used by both xlsx and CSV paths:
      - Resolve cluster_name and datacenter_name columns from headers using CLUSTER_ALIASES and DATACENTER_ALIASES (empty required sets).
      - Accumulate per-scope sub-aggregates during the row loop.
      - After the loop, build detectedScopes, scopeLabels, rawByScope identically to rvtoolsParser.
      - The aggregate() return type must include these three fields.
      - For parseXlsx(), when ESX Hosts data is available, attach it only to the top-level return (not to per-scope rawByScope entries — scope-level ESX data cannot be inferred without host-to-cluster mapping).
    The existing return shape (totalVcpus, totalVms, totalDiskGb, avgRamPerVmGb, vmCount, warnings) is unchanged.

    Step 5 — Create scopeAggregator.ts (src/lib/utils/import/scopeAggregator.ts):
    Export a single pure function:
      aggregateScopes(
        rawByScope: Map<string, Omit<ClusterImportResult, 'sourceFormat' | 'detectedScopes' | 'scopeLabels' | 'rawByScope'>>,
        selectedKeys: string[],
      ): Omit<ClusterImportResult, 'sourceFormat' | 'detectedScopes' | 'scopeLabels' | 'rawByScope'>
    Logic:
      - Filter rawByScope to selectedKeys only.
      - Sum totalVcpus, totalVms, totalDiskGb across selected scopes.
      - Recompute avgRamPerVmGb = (sum of avgRamPerVmGb * vmCount across scopes) / total vmCount. Handle zero-VM edge case (return 0).
      - vmCount = sum of vmCount across selected scopes.
      - warnings = flattened array of all warnings from selected scopes.
      - ESX fields (totalPcores, existingServerCount, socketsPerServer, coresPerSocket, ramPerServerGb, cpuUtilizationPercent, ramUtilizationPercent): copy from the first selected scope that has them defined (or omit if none).
      - If selectedKeys is empty, return zeroed result: { totalVcpus: 0, totalVms: 0, totalDiskGb: 0, avgRamPerVmGb: 0, vmCount: 0, warnings: [] }.
