**Step 1: Extend scenarioSchema (src/schemas/scenarioSchema.ts)**

Add 9 new optional fields to the `scenarioSchema` z.object:

1. `vsanFttPolicy`: Use `z.preprocess((v) => (v === '' ? undefined : v), z.enum(['mirror-1', 'mirror-2', 'mirror-3', 'raid5', 'raid6']).optional())` — empty string from `<select>` maps to undefined (legacy path per VSAN-12).

2. `vsanCompressionFactor`: Use `z.preprocess(numericPreprocess, z.union([z.literal(1.0), z.literal(1.3), z.literal(1.5), z.literal(2.0), z.literal(3.0)]).optional())` — `<select>` emits string, `numericPreprocess` converts to number.

3. `vsanSlackPercent`: `z.preprocess(numericPreprocess, z.number().min(0).max(100).optional())` — optional, `constraints.ts` null-coalesces to 25%.

4. `vsanCpuOverheadPercent`: `z.preprocess(numericPreprocess, z.number().min(0).max(100).optional())` — optional, constraints null-coalesces to 10%.

5. `vsanMemoryPerHostGb`: `z.preprocess(numericPreprocess, z.number().min(0).optional())` — optional, constraints null-coalesces to 6 GB.

6. `vsanVmSwapEnabled`: `z.boolean().optional().default(false)`.

7. `cpuGrowthPercent`: `z.preprocess(numericPreprocess, z.number().min(0).max(200).optional())`.

8. `memoryGrowthPercent`: `z.preprocess(numericPreprocess, z.number().min(0).max(200).optional())`.

9. `storageGrowthPercent`: `z.preprocess(numericPreprocess, z.number().min(0).max(200).optional())`.

Do NOT add `.default()` to vsanSlackPercent/vsanCpuOverheadPercent/vsanMemoryPerHostGb — keep them optional so absent = legacy path. Use `placeholder` attributes in the Input fields instead.
