2. Create `src/lib/utils/chartCapture.ts` with a single exported async function:

   ```typescript
   export function chartRefToDataUrl(
     container: HTMLDivElement | null,
   ): Promise<{ dataUrl: string; width: number; height: number } | null>
   ```

   Implementation:
   - If `container` is null or `container.querySelector('svg')` returns null, resolve with `null`.
   - Use the same XMLSerializer pipeline as `downloadChartPng.ts`:
     - `new XMLSerializer().serializeToString(svg)` -> Blob -> `URL.createObjectURL` -> `new Image()`.
     - On `img.onload`: create a canvas at 2x resolution (`svg.clientWidth * 2`, `svg.clientHeight * 2`), `ctx.scale(2, 2)`, fill white background, `ctx.drawImage(img, 0, 0)`, `URL.revokeObjectURL(url)`, resolve with `{ dataUrl: canvas.toDataURL('image/png'), width: svg.clientWidth, height: svg.clientHeight }`.
     - On `img.onerror`: `URL.revokeObjectURL(url)`, resolve with `null`.
   - Return a Promise wrapping the img.onload callback.
   - Return the LOGICAL pixel dimensions (svg.clientWidth/clientHeight), NOT the canvas dimensions (which are 2x). The consumer (exportPdf/exportPptx) will use these for scaling.

3. Create `src/lib/utils/__tests__/chartCapture.test.ts`:
   - Test: returns null when container is null
   - Test: returns null when container has no SVG child
   - (Cannot test actual SVG rendering in jsdom -- add a comment noting this is browser-tested via the export flow)
