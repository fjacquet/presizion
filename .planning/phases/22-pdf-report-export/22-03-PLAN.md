**Implementation details:**

1. **Lazy-load** pptxgenjs via `const PptxGenJS = (await import('pptxgenjs')).default`. pptxgenjs uses a default export.

2. **Capture chart images** BEFORE building slides. Same approach as exportPdf: for each scenario, capture `capacity-${scenario.id}` and `minnodes-${scenario.id}` via `chartRefToDataUrl`. Use `Promise.all`. Handle null gracefully (skip chart on slide if unavailable).

3. **Create presentation:**
   ```typescript
   const pptx = new PptxGenJS()
   pptx.layout = 'LAYOUT_WIDE'  // 13.33" x 7.5"
   pptx.author = 'Presizion'
   pptx.title = 'Cluster Sizing Report'
   ```

4. **Slide 1: Title slide**
   - "Presizion" as title text (fontSize 36, bold, color '3B82F6').
   - "Cluster Sizing Report" subtitle (fontSize 24).
   - "Generated: {date}" smaller text (fontSize 14, color '6B7280').
   - Cluster summary: "{cluster.totalVms} VMs | {cluster.totalVcpus} vCPUs | {cluster.totalPcores} pCores" (fontSize 12).

5. **Slide 2: Executive Summary table**
   - Title: "Executive Summary" (fontSize 24, bold).
   - Table with header row (blue background '#3B82F6', white text) and data rows.
   - Columns: Scenario | Servers | Limiting Resource | CPU Util % | RAM Util %.
   - Use pptxgenjs `slide.addTable(rows, opts)` where rows is `TableRow[][]`. Each cell is `{ text: string, options?: { bold?: boolean, fill?: { color: string }, color?: string } }`.

6. **Per-scenario slides** (loop over scenarios):
   - **Slide: Capacity Breakdown Table** for scenario N.
     - Title: "Capacity Breakdown -- {scenario.name}".
     - Table: Resource | Required | Spare | Excess | Total Configured.
     - Rows: CPU GHz, Memory GiB, Raw Storage TiB (divide storage values by 1024).
   - **Slide: Capacity Charts** for scenario N (if chart images were captured).
     - Title: "Capacity Charts -- {scenario.name}".
     - Capacity stacked chart image: position at x=0.5, y=1.2, scaled to fit ~12" wide. Maintain aspect ratio: `h = (imgData.height / imgData.width) * 12`.
     - Min nodes chart image: below capacity chart. If both fit on one slide (~7.5" total height), place on same slide. Otherwise, add a second slide.
     - Use `slide.addImage({ data: imgData.dataUrl, x, y, w, h })`. The `data` property accepts a data URL string directly.

7. **Final slide: Comparison Table**
   - Title: "Scenario Comparison".
   - Table with columns: Metric | As-Is | Scenario1 | Scenario2 | ...
   - Rows: Servers Required, Server Config, Total pCores, Limiting Resource, vCPU:pCore Ratio, VMs/Server, CPU Util %, RAM Util %, Disk Util %.
   - Use smaller font (fontSize 10) to fit more data.

8. **Save:** `await pptx.writeFile({ fileName: 'presizion-sizing-report.pptx' })`.
