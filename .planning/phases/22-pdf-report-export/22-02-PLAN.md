Create `src/lib/utils/exportPdf.ts` exporting a single async function:

```typescript
export async function exportPdf(
  cluster: OldCluster,
  scenarios: readonly Scenario[],
  results: readonly ScenarioResult[],
  breakdowns: readonly VsanCapacityBreakdown[],
  chartRefs: Record<string, HTMLDivElement | null>,
): Promise<void>
```

**Implementation details:**

1. **Lazy-load** both jsPDF and jspdf-autotable via `Promise.all([import('jspdf'), import('jspdf-autotable')])`. Use named imports: `{ jsPDF }` from jspdf and `{ autoTable }` from jspdf-autotable (v5 standalone function pattern, NOT the prototype method).

2. **Capture chart images** BEFORE building PDF content. For each scenario, capture:
   - `chartRefs[`capacity-${scenario.id}`]` via `chartRefToDataUrl`
   - `chartRefs[`minnodes-${scenario.id}`]` via `chartRefToDataUrl`
   Use `Promise.all` to capture all charts. Handle null gracefully (skip chart in PDF if not available).

3. **Create jsPDF document:** `new jsPDF({ orientation: 'portrait', unit: 'mm', format: 'a4' })`.
   Constants: `PAGE_W = doc.internal.pageSize.getWidth()` (210mm), `PAGE_H = doc.internal.pageSize.getHeight()` (297mm), `MARGIN = 14`.
   Track y-cursor: `let y = MARGIN`.
   Helper: `ensureSpace(needed: number)` -- if `y + needed > PAGE_H - MARGIN`, call `doc.addPage()` and reset `y = MARGIN`.

4. **Title page:**
   - "Presizion -- Cluster Sizing Report" centered at y=60, font size 22, bold.
   - "Generated: {date}" at y=75, font size 12, normal.
   - Cluster summary: Total VMs, Total vCPUs, Total pCores as a small table or text lines.
   - `doc.addPage()`.

5. **Executive summary table** (one row per scenario):
   - Section heading "Executive Summary" (font 14, bold).
   - `autoTable(doc, { ... })` with columns: Scenario | Servers Required | Limiting Resource | CPU Util % | RAM Util %.
   - Theme 'grid', headStyles blue (fillColor [59, 130, 246]).
   - Get finalY via `(doc as { lastAutoTable?: { finalY?: number } }).lastAutoTable?.finalY ?? y + 20`.

6. **Per-scenario sections** (loop over scenarios):
   - Section heading "Scenario: {name}" (font 14, bold). `ensureSpace(60)`.
   - **Capacity breakdown table** via autoTable:
     - Columns: Resource | Required | Spare | Excess | Total Configured
     - Rows: CPU GHz, Memory GiB, Raw Storage TiB (storage values / 1024).
     - Theme 'striped', headStyles gray ([107, 114, 128]).
     - First column bold, cellWidth 40.
   - **Chart images** (if captured):
     - Capacity chart: scale to fit page width (`maxW = PAGE_W - 2 * MARGIN`). Convert pixel width to mm proportionally. Use `doc.addImage(dataUrl, 'PNG', MARGIN, y, maxW, imgH, 'capacity-chart-{i}', 'FAST')`. Always pass unique alias (7th param) to prevent blank-page bug.
     - Min nodes chart: same approach with alias `'minnodes-chart-{i}'`.
     - Scaling: `mmPerPx = (PAGE_W - 2 * MARGIN) / imgData.width`. `imgH = imgData.height * mmPerPx`.

7. **Comparison table** (after all scenarios):
   - `ensureSpace(40)`. Section heading "Scenario Comparison".
   - autoTable with columns: Metric | As-Is | Scenario1 | Scenario2 | ...
   - Rows mirror ComparisonTable.tsx: Servers Required, Server Config, Total pCores, Limiting Resource, vCPU:pCore Ratio, VMs/Server, Headroom %, CPU Util %, RAM Util %, Disk Util %.
   - Pull data from cluster, scenarios, and results arrays.

8. **Save:** `doc.save('presizion-sizing-report.pdf')`.

**Type safety:**
- Cast `doc` for lastAutoTable access: `(doc as { lastAutoTable?: { finalY?: number } }).lastAutoTable?.finalY`.
- No `any` types. Use proper imports for OldCluster, Scenario, ScenarioResult, VsanCapacityBreakdown.
