<objective>
Fix import scoping so that ESX host config (RAM/server, sockets, cores, pCores, server count)
and ESX Performance data (CPU/RAM utilization) are stored per cluster scope, not globally.

Purpose: When a user selects a specific cluster in the scope filter, the sizing engine must
receive that cluster's actual host configuration -- not the first host row from the entire file.
This is a data-integrity bug (GH #4) that produces incorrect sizing results for multi-cluster imports.

Output: Parsers populate per-scope ScopeData with ESX fields; aggregateScopes sums additive fields
and warns on heterogeneous RAM; RVTools vHost extracts sockets/cores/RAM; all changes tested.
</objective>
