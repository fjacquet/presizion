Make four changes to WizardShell.tsx:

**1. NAV-01 — Hide tagline on mobile** (line 75-77):
Change the tagline `<p>` element from:
```tsx
<p className="text-sm text-muted-foreground mt-1">
```
to:

**2. MOBILE-03 — Toolbar touch targets** (lines 52-65):
- Reset button: Change from `<Button variant="ghost" size="sm" ...>` to add explicit 44px sizing. Replace `size="sm"` with `size="icon"` and add `className="h-11 w-11 p-0"`:
```tsx
<Button variant="ghost" size="icon" onClick={() => setResetOpen(true)} aria-label="Reset" className="h-11 w-11 p-0">
  <RotateCcw className="h-4 w-4" />
</Button>
```

- Store-Predict link: Change `h-8 w-8` to `h-11 w-11`:
```tsx
<a
  href={STORE_PREDICT_URL}
  target="_blank"
  rel="noopener noreferrer"
  title="Storage Calculator (Store-Predict)"
  className="inline-flex items-center justify-center rounded-md h-11 w-11 text-muted-foreground hover:text-foreground hover:bg-accent transition-colors"
>
```

- ThemeToggle: The ThemeToggle component uses `size="icon"` which is `size-8` (32px). Add a wrapper div with 44px sizing. In WizardShell, change:
```tsx
<div className="absolute right-0 top-0">
  <ThemeToggle />
</div>
```
to:
```tsx
<div className="absolute right-0 top-0 [&_button]:h-11 [&_button]:w-11">
  <ThemeToggle />
</div>
```
This applies h-11 w-11 to the Button inside ThemeToggle without modifying ThemeToggle.tsx itself.

**3. NAV-04 — Sticky Back/Next navigation on mobile** (lines 91-102):
Replace the Back/Next navigation block entirely:
```tsx
{currentStep > 1 && (
  <div
    className="sticky bottom-0 z-10 bg-background/95 backdrop-blur border-t px-4 py-3 flex justify-between print:hidden sm:static sm:bg-transparent sm:backdrop-blur-none sm:mt-8 sm:pt-4 sm:z-auto"
    style={{ paddingBottom: 'calc(0.75rem + env(safe-area-inset-bottom, 0px))' }}
  >
    <Button type="button" variant="outline" onClick={prevStep} className="min-h-[44px]">
      Back
    </Button>
    {currentStep === 2 && (
      <Button type="button" onClick={nextStep} className="min-h-[44px]">
        Next: Review &amp; Export
      </Button>
    )}
  </div>
)}
```

Key changes:
- `sticky bottom-0 z-10` — sticks to bottom on mobile so user never has to scroll down to find nav
- `bg-background/95 backdrop-blur` — semi-transparent with blur effect on mobile
- `sm:static sm:bg-transparent sm:backdrop-blur-none sm:mt-8 sm:pt-4 sm:z-auto` — on desktop (>=640px), reverts to original in-flow layout
- `paddingBottom: calc(0.75rem + env(safe-area-inset-bottom, 0px))` — safe-area for iPhone home indicator
- `min-h-[44px]` on both buttons for MOBILE-03 touch targets

**4. Main element padding for sticky nav clearance** (line 85):
Change:
```tsx
<main>
```
to:
```tsx
<main className={currentStep > 1 ? 'pb-20 sm:pb-0' : ''}>
```
This adds bottom padding only when sticky nav is visible (steps 2 and 3) to prevent content from being hidden behind the sticky bar. On desktop (`sm:pb-0`) no extra padding is needed since nav is in-flow.
