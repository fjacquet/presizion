2. **MOBILE-01 — iOS auto-zoom prevention**: Add a new `@media (hover: none)` block AFTER the existing `body` rule block (before the `h1, h2` heading rules). This targets touch-primary devices (phones, tablets) and sets all form inputs to 16px font-size, which is the minimum iOS Safari requires to not auto-zoom on focus:
```css
/* MOBILE-01: prevent iOS Safari auto-zoom on input focus */
@media (hover: none) {
  input,
  input[type="number"],
  input[type="text"],
  input[type="search"],
  input[type="email"],
  select,
  textarea {
    font-size: 16px !important;
  }
}
```

Use `@media (hover: none)` (not `max-width: 639px`) because touch devices are identified by absence of hover capability, not screen width. A desktop with a touch screen should still zoom normally.

Do NOT modify the `@layer base` block, `@theme inline` block, `@media print` block, dark mode `:root` block, or `.dark` class. Only modify `body` and add the new media query block.

  <action>
Modify the outermost `<div>` in WizardShell.tsx (currently line 49) to replace `min-h-screen` with dvh viewport height and add overflow containment:

**Before:**
```tsx
<div className="min-h-screen bg-background">
```

**After:**
```tsx
<div className="bg-background overflow-x-hidden" style={{ minHeight: '100dvh' }}>
```

Rationale:
- `min-h-screen` generates `min-height: 100vh` which does not account for the iOS Safari dynamic address bar height. `100dvh` (dynamic viewport height) tracks the actual visible area as the address bar shows/hides.
- Using `style={{ minHeight: '100dvh' }}` instead of Tailwind `min-h-[100dvh]` ensures a clean fallback: browsers that do not support `dvh` (Safari < 15.4) will ignore the property and the `#root` element's existing `min-height: 100svh` in index.css provides the fallback.
- `overflow-x-hidden` class is belt-and-suspenders with the `body` overflow rule from Task 1 — it prevents any child content from creating horizontal scroll at the shell level.

Do NOT change any other elements in WizardShell.tsx in this task — header toolbar changes, sticky nav, and touch targets are handled in Plan 02.
