    **Step 3 — Create manifest (GREEN - part 2).** Write `public/manifest.webmanifest`:
    ```json
    {
      "name": "Presizion \u2014 Cluster Refresh Sizing",
      "short_name": "Presizion",
      "description": "Size a refreshed server cluster from existing environment metrics",
      "start_url": "/presizion/",
      "scope": "/presizion/",
      "display": "standalone",
      "background_color": "#ffffff",
      "theme_color": "#3B82F6",
      "icons": [
        {
          "src": "/presizion/icons/pwa-192x192.png",
          "sizes": "192x192",
          "type": "image/png"
        },
        {
          "src": "/presizion/icons/pwa-512x512.png",
          "sizes": "512x512",
          "type": "image/png"
        },
        {
          "src": "/presizion/icons/maskable-512x512.png",
          "sizes": "512x512",
          "type": "image/png",
          "purpose": "maskable"
        }
      ]
    }
    ```

    CRITICAL: All `src` paths use absolute `/presizion/` prefix. `start_url` and `scope` both use `/presizion/`. This is mandatory for GitHub Pages subpath deployment (M-3 pitfall from research).

    **Step 4 — Update index.html (GREEN - part 3).** Add meta tags to the `<head>` section of `index.html`. The updated head should contain (in order):

    1. Existing: `<meta charset="UTF-8" />`
    2. Existing: `<link rel="icon" type="image/svg+xml" href="/favicon.svg" />`
    3. UPDATED: `<meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover" />` (add `viewport-fit=cover`)
    4. Existing: `<title>Presizion — Cluster Refresh Sizing</title>`
    5. NEW: `<link rel="manifest" href="/presizion/manifest.webmanifest" />`
    6. NEW: `<meta name="apple-mobile-web-app-capable" content="yes" />`
    7. NEW: `<meta name="apple-mobile-web-app-status-bar-style" content="default" />`
    8. NEW: `<meta name="apple-mobile-web-app-title" content="Presizion" />`
    9. NEW: `<link rel="apple-touch-icon" sizes="180x180" href="/presizion/icons/apple-touch-icon-180x180.png" />`
    10. NEW: `<meta name="theme-color" content="#ffffff" media="(prefers-color-scheme: light)" />`
    11. NEW: `<meta name="theme-color" content="#0a0a0a" media="(prefers-color-scheme: dark)" />`
    12. Existing: theme init script block (keep as-is)

    Do NOT use `apple-mobile-web-app-status-bar-style: black-translucent` (M-5 pitfall -- causes content under notch). Use `"default"`.
