### Sizing Calculations

- [x] **CALC-01**: App calculates CPU-limited server count: `ceil((totalVcpus × headroom) / targetVcpuToPCoreRatio / coresPerServer)` — Phase 1 ✓
- [x] **CALC-02**: App calculates RAM-limited server count: `ceil((totalVms × ramPerVmGb × headroom) / ramPerServerGb)` — Phase 1 ✓
- [x] **CALC-03**: App calculates disk-limited server count: `ceil((totalVms × diskPerVmGb × headroom) / diskPerServerGb)` — Phase 1 ✓
- [x] **CALC-04**: When N+1 HA reserve is enabled, app adds 1 to the final server count after constraint resolution — Phase 1 ✓
- [x] **CALC-05**: App identifies the final server count (max of CPU/RAM/disk constraints) and clearly labels the limiting resource — Phase 1 ✓
- [x] **CALC-06**: App computes achieved vCPU:pCore ratio, VMs per server, and estimated CPU/RAM/disk utilization percentages for each scenario — Phase 1 ✓
- [x] **CALC-07**: Each key output value displays its formula and the specific input parameters used, inline in the UI — Phase 1/4 ✓

## Out of Scope (carried forward)

| Feature | Reason |
|---------|--------|
| Pre-defined server SKU library | Avoids maintaining a hardware catalog; user-input only keeps the tool vendor-neutral |
| Backend, API, auth, or user accounts | Client-side only by design; no server infrastructure |
| TCO/ROI or pricing/BOM calculations | Capacity sizing only in v1 |
| Per-VM-level workload modeling | Aggregate-based calculations only |
| vCenter, CloudIQ, or external system integration | No backend to proxy requests |
| PWA / offline capability | Basic static hosting is sufficient |
| Localization / i18n | English only for v1 |
| localStorage persistence (PERS-01) | Deferred from v1.0; not included in v1.2 |
| URL hash state sharing (PERS-02) | Deferred from v1.0 |
| Explicit light/dark mode toggle (UI-01) | OS preference sufficient for v1.x |
