## Milestone Summary

**Key Accomplishments:**

1. Complete sizing engine — CPU/RAM/disk + SPECint mode + utilization right-sizing, 254 Vitest tests
2. 3-step wizard — React + Tailwind v4 + shadcn/ui with live calculations, inline formula display, dark mode, GitHub Pages deployment
3. Side-by-side comparison report — As-Is/To-Be table with color-coded utilization, CSV/JSON/clipboard/PNG export, print/PDF layout
4. SPECint mode and utilization right-sizing — global toggle, conditional UI fields, delta formula
5. Bar chart visualization — Recharts BarChart comparing CPU/RAM/disk counts across scenarios with PNG download
6. File import — RVTools .xlsx and LiveOptics .zip/.xlsx/.csv auto-fill with preview/confirm modal

**Key Decisions:**

- Math before UI discipline: sizing library correctness established in Phase 1 before any UI component was written
- z.preprocess (not z.coerce.number) for numeric Zod fields — empty string inputs produce ZodError, protecting form validation
- derive-on-read: ScenarioResult never stored in Zustand — useScenariosResults derives on demand
- xlsx@0.18.5 (last MIT-licensed SheetJS) for file import — locked version to avoid license issues
- Dynamic import() for xlsx/jszip — Vite code-splits both to lazy chunk (~900KB SheetJS deferred)
- Wave 0 it.todo stubs (Nyquist pattern) used throughout phases 1-7; phases 8-10 as informal sprint

**Technical Debt Incurred:**

- Phases 8-10 implemented outside GSD framework (no Wave 0 stubs per GSD convention) — documented in Phase 8 backfill
- parseNumericInput.ts dead code existed from Phase 1 through Phase 7 — deleted in Phase 8

**Issues Resolved:**

- ResizeObserver missing in jsdom (Recharts dependency) — solved by mocking in test-setup.ts
- @base-ui/react/dialog import: `{ Dialog }` not `* as Dialog` — corrected during Phase 10
- exactOptionalPropertyTypes TS flag requiring conditional spreads for optional numeric fields
